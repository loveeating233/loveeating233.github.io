this 指向

发表于2023-03-17|更新于2023-05-01
|阅读量:

默认绑定规则：window

函数的独立调用：window

你可以理解为 window.test()

立即执行函数： window

在浏览器环境中，立即执行函数的 this 指向 window。可能以后在模块中指向的是模块中的环境.如果换一个环境，它指向的是相应的全局对象

闭包

你可以理解为是函数独立调用

隐式绑定：谁调用，this 指向谁

只有函数执行，里面的 this 才有指向，函数不执行，里面的 this 毫无意义。

下图中，这两个 this 是不一样的。每一个函数的执行都会有一个自身的 this 指向，虽然可能它们指向的是同一个东西，但每个 this 本身是不一样的

上面一个 this 是 obj，因为是 obj 调用

下面一个是 window，因为自调用(函数独立调用)

看一道题目

函数中的 this 只有函数被调用的时候，才会有指向，因为是 obj 调用的，所以 this 的指向是 obj

调用,所以是 window
